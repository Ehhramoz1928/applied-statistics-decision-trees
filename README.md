This project establishes a robust end-to-end data science pipeline in R to address critical risk assessments in financial lending and macroeconomic forecasting. To ensure model generalizability and eliminate data leakage, a consumer credit dataset tracking binary credit card defaults was partitioned into an independent training-to-validation split, while an economic sequence tracking continuous wage growth utilized a separate isolated partition. Initial baseline models exhibited high variance, which was systematically mitigated by implementing cost-complexity pruning with an optimized parameter threshold to strip away overfitting statistical noise. The resulting classification tree optimized consumer credit default predictions, generating a validation confusion matrix to deliver concrete metrics for institutional risk management. Simultaneously, the pruned regression tree minimized numeric variance for economic forecasting, achieving highly stable validation performance metrics to ensure predictions deviated minimally from observed actuals. Ultimately, this framework successfully uncovered non-linear tipping points, providing stakeholders and corporate planners with actionable, data-driven benchmarks to anticipate sharp shifts in operational budgets.

At its core, a decision tree is a non-parametric supervised learning algorithm that segments the feature space into a set of non-overlapping regions. The process is fundamentally top-down and greedy, beginning at a single root node that represents the entire, unpartitioned population based on the feature providing the best initial split. From there, internal split nodes bifurcate the data based on specific feature thresholds, continuing down to terminal leaf nodes. These leaves represent the final subgroups containing the pooled data points where no further splits occur, assigning a majority class for classification tasks or a localized mathematical average for regression tasks. The algorithm is classified as greedy because at every individual stage, it selects the absolute best local split available without considering whether a different split might lead to a superior overall tree multiple steps down the line.

To determine the optimal boundary at any internal node, the algorithm evaluates how homogeneous or pure the resulting sub-nodes will be, using entirely different mathematical metrics based on the target variable type. For classification trees, the objective is to maximize node homogeneity to ensure each leaf contains mostly one class, evaluating partitions via Gini Impurity or Entropy and Information Gain to measure the level of disorder or unpredictability. Conversely, regression trees aim to group data points with similar continuous numeric values together. This is achieved by minimizing variance, where the algorithm evaluates every possible split across continuous features and selects the partition that minimizes the sum of squared residuals within the resulting child nodes.

A major structural vulnerability of decision trees is their high variance, as an unrestricted tree will continue splitting the data recursively until every individual training observation resides in its own isolated leaf node. While this yields a near-perfect training error, it severely overfits to statistical noise and generalizes poorly to unseen validation data. To balance this bias-variance tradeoff, cost-complexity pruning is applied by allowing the tree to grow fully and then systematically stripping back lower-level splits. This process is governed by a complexity parameter that penalizes the model for having too many terminal nodes, minimizing a cost function that mathematically weighs residual error against overall structural complexity. This optimization collapses weaker nodes that do not significantly contribute to reducing model error, resulting in a streamlined, highly interpretable, and generalizable structure.
